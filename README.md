# Local Finance BI Dashboard

一个本地运行的个人收支 BI 看板，用来导入信用卡账单、未出账明细和借记卡流水，按自然月统计收入支出，并对大额消费、收入来源和月度趋势做可视化分析。

项目面向个人本地使用：账单文件只在浏览器中解析，分析数据保存在本机浏览器 IndexedDB，不需要后端服务，也不会上传到云端。

## 当前能力

- 导入 `.xls`、`.xlsx`、`.csv` 格式的信用卡账单、未出账明细和借记卡流水。
- 自动识别信用卡 / 借记卡账单表头，导入前展示预览。
- 支持一键加载完全虚构的演示数据，用于向别人展示看板功能，不需要暴露真实账单。
- 按自然月统计总收入、总支出、信用卡支出、借记卡支出和净结余。
- 全局筛选支持月份下拉、快捷时间范围、来源、账户、卡片类型、交易类型、分类、金额区间和关键词。
- 月度收入 / 支出图支持坐标轴、金额刻度和悬停金额提示。
- 净结余趋势图支持横纵坐标轴、金额刻度、0 轴和悬停金额提示。
- 月度对比矩阵展示每月收入、支出、信用卡支出、借记卡支出、净结余和交易数。
- `TOP N` 消费支持指定月份、多月份对比、近 3 月、近 6 月、全部月份、金额区间、来源、账户、卡片类型、分类、关键词和排序方式。
- `TOP N` 收入支持和消费模块一致的筛选与对比能力。
- 支持分类支出结构、来源支出结构、收入分类结构和收入来源结构。
- 支持筛选后的交易明细表，方便下钻检查具体交易。
- 支持交易分类手动修正，并可保存为后续自动分类规则。
- 支持导出月度汇总 CSV、导出当前筛选后的 TOP N 消费 / 收入 CSV、备份 / 恢复本地 JSON 数据。

## 本地运行

需要先安装 Node.js 和 npm。建议使用 Node.js 18 或更高版本。

```bash
git clone https://github.com/fengyuanshen/local-finance-bi-dashboard.git
cd local-finance-bi-dashboard
npm install
npm run dev
```

启动后打开终端显示的本地地址，通常是：

```text
http://127.0.0.1:5173/
```

从 GitHub 下载源码后不能直接双击运行，因为依赖不会提交到仓库。需要先执行 `npm install` 安装依赖，再执行 `npm run dev` 启动本地服务。

## Demo 展示模式

如果要向别人演示报表功能，但不想使用自己的真实交易明细，可以在页面的“导入中心”或“数据管理”中点击：

- `加载演示数据`
- `清空演示数据`

演示数据由应用在本地生成，包含 2025 年 7 月到 2026 年 5 月的虚构工资、奖金、信用卡消费、借记卡消费、退款和信用卡还款转账。所有公司、商户、账号、卡号、交易摘要和金额组合均为模拟内容，不包含任何真实个人信息。

加载演示数据会替换当前本地交易和分类规则；如果浏览器里已有非演示数据，页面会先弹出确认提示。`清空演示数据` 只会删除带有演示标记的数据，不会删除你后续重新导入的真实账单。需要彻底清空本地数据时，可以使用 `清空本地数据`。

## 统计口径

- 月度统计按自然月计算，不按信用卡账单周期。
- 信用卡消费以交易日期归属月份。
- 借记卡 `存入金额` 计为收入，`取出金额` 计为支出。
- 信用卡还款、账户互转、理财申购赎回默认识别为转账，不进入消费支出。
- 退款会从消费类统计中排除，并按退款记录处理。
- `TOP N` 消费按单笔支出金额从高到低排序，默认 TOP 10，可切换为日期降序。
- `TOP N` 收入基于 `income > 0` 的交易，默认排除转账类记录。
- 全局金额筛选会匹配交易的收入或支出金额；TOP 模块的金额筛选只作用于对应模块的收入或支出金额。

## 数据隐私

- 项目没有后端服务。
- 导入的账单文件在浏览器本地解析。
- 分析后的交易数据保存在浏览器 IndexedDB。
- 如果清空浏览器站点数据，本地保存的看板数据也会被清除。
- 不建议把真实账单文件、导出的备份 JSON 或任何个人敏感数据提交到 Git 仓库。

## 依赖说明

项目依赖不会提交到 GitHub。其他人克隆仓库后，需要运行：

```bash
npm install
```

这会根据 `package.json` 和 `package-lock.json` 下载依赖并生成本地 `node_modules/` 目录。

其中 `xlsx` 用于解析 Excel / 老式 `.xls` 银行账单。npm audit 对该包有已知安全提示，且 npm 当前没有可用修复版本。由于本项目是本地工具，建议只导入来自银行、且你确认可信的账单文件。

## Git 忽略项

仓库不会提交以下内容：

- `node_modules/`
- `dist/`
- `.DS_Store`
- 日志文件
- `.env`

# Local Finance BI Dashboard

一个本地运行的信用卡 / 借记卡收支分析看板，用来导入银行账单、按自然月统计收入支出、查看每月大额消费，并做多月份横向对比。

项目当前面向个人本地使用：账单文件只在浏览器中解析，分析数据保存在本机浏览器 IndexedDB，不需要后端服务，也不会上传到云端。

## 功能特性

- 导入 `.xls`、`.xlsx`、`.csv` 格式的信用卡账单、未出账明细和借记卡流水。
- 自动识别信用卡 / 借记卡账单表头，并在导入前展示预览。
- 按自然月统计总收入、总支出、信用卡支出、借记卡支出和净结余。
- 支持月度收入 / 支出图、净结余趋势图和月度对比矩阵。
- 每月 `TOP N` 消费可自定义数量，并支持选择指定月份、近 3 月、近 6 月或全部月份做横向对比。
- 提供分类支出结构、来源支出结构和每月大额消费明细。
- 支持交易分类手动修正，并可保存为后续自动分类规则。
- 支持导出月度汇总 CSV、导出 TOP N 明细 CSV、备份 / 恢复本地 JSON 数据。

## 本地运行

需要先安装 Node.js 和 npm。建议使用 Node.js 18 或更高版本。

```bash
git clone https://github.com/fengyuanshen/local-finance-bi-dashboard.git
cd local-finance-bi-dashboard
npm install
npm run dev
```

启动后打开终端显示的本地地址，通常是：

```text
http://127.0.0.1:5173/
```

如果只是从 GitHub 下载代码，不能直接双击源码运行。需要先执行 `npm install` 安装依赖，再执行 `npm run dev` 启动本地开发服务。

## 支持的账单字段

信用卡账单字段：

- 交易日期
- 记账日期
- 交易摘要
- 卡号末四位
- 卡片类型
- 交易币种
- 交易金额
- 原始交易金额&币种

借记卡账单字段：

- 账号
- 账户类型
- 日期范围
- 币别号
- 序号
- 交易日期
- 交易类型
- 存入金额
- 取出金额
- 余额
- 对方全称
- 交易对手账号
- 附言

## 统计口径

- 月度统计按自然月计算，不按信用卡账单周期。
- 信用卡消费以交易日期归属月份。
- 借记卡 `存入金额` 计为收入，`取出金额` 计为支出。
- 信用卡还款、账户互转、理财申购赎回默认识别为转账，不进入消费支出。
- 退款会从消费类统计中排除，并按退款记录处理。
- `TOP N` 消费按单笔支出金额从高到低排序，默认 TOP 10。

## 数据隐私

- 项目没有后端服务。
- 导入的账单文件在浏览器本地解析。
- 分析后的交易数据保存在浏览器 IndexedDB。
- 如果清空浏览器站点数据，本地保存的看板数据也会被清除。
- 不建议把真实账单文件、导出的备份 JSON 或任何个人敏感数据提交到 Git 仓库。

## 依赖说明

项目依赖不会提交到 GitHub。其他人克隆仓库后，需要运行：

```bash
npm install
```

这会根据 `package.json` 和 `package-lock.json` 下载依赖并生成本地 `node_modules/` 目录。

其中 `xlsx` 用于解析 Excel / 老式 `.xls` 银行账单。npm audit 对该包有已知安全提示，且 npm 当前没有可用修复版本。由于本项目是本地工具，建议只导入来自银行、且你确认可信的账单文件。

## Git 忽略项

仓库不会提交以下内容：

- `node_modules/`
- `dist/`
- `.DS_Store`
- 日志文件
- `.env`
